## 1. 题目要求 

**通用技术栈**：Spring Boo 3 + MyBatis-Plus / Spring Data JPA + MySQL 8.0 + Redis + + Vue.js 3 

**通用要求**：实现基础的用户登录/注册、权限控制（至少区分`普通用户`和`管理员`）、API接口文档（Swagger/knife4j）。

**公平性评分标准（每个系统均适用）：**

1. **基础功能完备性 (40%)**：完成所有“核心功能”列表中的条目。
2. **Redis缓存应用合理性 (30%)**：按要求实现指定的缓存应用点，并处理了缓存一致性。
3. **代码质量与架构 (20%)**：代码结构清晰，遵循分层架构，有基本的异常处理和日志。
4. **前端交互与体验 (10%)**：页面布局合理，能通过前端调用完成所有核心操作。

## 2.题目详情

- **核心功能 (至少完成)**：

1. **文档管理**：树形目录管理文档，支持Markdown编辑器。
2. **版本历史**：保存文档的修改历史，支持版本对比和回滚。
3. **全文检索**：集成Elasticsearch或使用数据库`LIKE`实现简单的标题/内容搜索。
4. **文档权限**：可设置文档的查看/编辑权限（公开、部门可见、私有）。

- **Redis缓存应用**：

1. **缓存热门/最新文档**：将“最近更新”和“最多阅读”的文档列表缓存。
2. **文档内容缓存**：将已发布的文档最终渲染内容（HTML）缓存，极大提升访问速度。
3. **用户编辑锁**：用户开始编辑文档时，在Redis设置一个`editing:docId`的键，防止多人同时编辑产生覆盖，编辑完成或离开页面时释放。

## 3.项目细节

- 功能、业务逻辑和前端风格等请仿照语雀来写（不要阉割版）
- 请给出完善的配套说明文档
- 实现双端（分为用户端和管理员后台管理端）
- 前端注意风格统一，尽量复用同一UI来实现风格统一
- 支持暗黑模式和浅色模式
- 支持回收站
- 但一个文件彻底被删除时，保留在本地的对应文件也应该被删除，防止内存爆炸
- 支持分享功能
- 注册支持邮箱或者手机号注册发送六位验证码，不过如果没配置邮箱和邮箱授权码，就弹出提示框，说是测试状态下直接给出验证码，如要体验完整功能，请补充邮箱和邮箱授权码
- 注册时请注意邮箱，手机号，用户名都各自唯一（但手机号可绑定邮箱，反之依然）
- 支持修改用户信息
- 用雪花算法生成唯一id，但要注意前端javascrip精度丢失问题
- 请完善提示框功能，避免直接报错
- 用户可查看/回滚到任意历史版本（类似 Git 的 commit）
- 自动保存草稿（每 30 秒）
- 支持多人同时编辑同一文档（基于 OT 或 CRDT 算法）
- 实时显示其他协作者光标位置与选中文本（带头像/姓名）
- 冲突自动合并 + 手动解决提示
- 用户可创建多个知识库（公开/私有/团队）
- 每个知识库有独立目录树（支持无限层级）
- 支持知识库成员邀请与权限控制（只读/编辑/管理员）
- 后端删除数据库记录后，**异步任务**清理本地/云存储文件
- 增加 `deleted_at` 软删除字段，回收站保留 30 天
- 彻底删除前二次确认：“此操作不可逆，是否继续？”
- 同一 IP / 手机号 / 邮箱 1 分钟内最多请求 1 次
- 记录关键操作：登录、删除、分享、权限变更
- 管理员后台可查日志（时间、IP、用户、操作）
- 所有弹窗、提示、加载状态风格一致
- 暗黑/浅色模式通过 CSS 变量切换，支持系统自动适配
- 后端返回的雪花 ID **必须以字符串形式传输**（JSON 中）
- 前端禁止将 ID 转为 Number（避免 > 2^53 -1 精度丢失）
- 示例：`{ id: "1823948129384712394" }` 而非 `{ id: 1823948129384712394 }`
- 所有 API 错误捕获后，**不直接 alert(error.message)**
- 映射为用户友好文案（如：“网络不稳，请重试”、“邮箱已被注册”）
- 提供“复制错误码”按钮供反馈
