# 三级权限模型实现总结

## 📋 项目概述

本次更新实现了企业内部知识库系统的三级权限模型（公司公开/部门/私有），并添加了完整的文档查看和修改记录功能，同时实现了日志自动清理机制（保留15天）。

## 🎯 核心功能

### 1. 三级权限模型

**知识库类型：**
- **COMPANY（公司公开）** - 全公司所有员工可见、可编辑
- **DEPARTMENT（部门）** - 部门内成员可见、可编辑，部门部长有管理权
- **PRIVATE（私有）** - 仅创建者可见、可编辑

**角色权限：**
- **系统管理员（ADMIN）** - 拥有所有权限
- **部门部长** - 可管理本部门知识库
- **普通员工（USER）** - 可访问公司公开、所属部门、自己的私有知识库

### 2. 文档操作记录

**查看记录（document_view_logs）：**
- 记录用户ID、用户名、IP地址、User-Agent
- 支持查看时长统计
- 自动保留15天

**修改记录（document_edit_logs）：**
- 记录操作类型（CREATE/UPDATE/DELETE/PUBLISH）
- 记录修改前后的标题和内容长度变化
- 记录提交说明（commit_message）
- 自动保留15天

### 3. 自动清理机制

**定时任务：**
- 每天凌晨3点自动执行
- 清理超过15天的查看记录和修改记录
- 使用批量删除，避免性能影响
- 记录清理日志便于监控

## 🔐 权限检查逻辑

### 知识库权限

**可见性检查（ensureKbVisible）：**
```
系统管理员 → 允许访问
↓
知识库类型？
├─ COMPANY → 允许访问
├─ DEPARTMENT → 同部门？→ 是：允许访问 / 否：拒绝访问
└─ PRIVATE → 是创建者？→ 是：允许访问 / 否：拒绝访问
```

**编辑权限检查（ensureKbEditor）：**
- COMPANY：所有人可编辑
- DEPARTMENT：同部门成员可编辑
- PRIVATE：仅创建者可编辑

**管理权限检查（ensureKbAdmin）：**
- 系统管理员：可管理所有知识库
- 部门部长：可管理本部门知识库
- 创建者：可管理自己创建的知识库

### 文档权限

文档权限继承所属知识库的权限规则，同时考虑文档的可见性设置（PUBLIC/TEAM/PRIVATE）。

## 📊 性能优化

### 索引设计

**新增索引：**
- `users.department_id` - 查询部门成员
- `departments.parent_id` - 查询子部门
- `departments.manager_id` - 查询部门部长管理的部门
- `document_view_logs.doc_id` - 查询文档查看记录
- `document_view_logs.user_id` - 查询用户查看记录
- `document_view_logs.created_at` - 按时间查询
- `document_edit_logs.doc_id` - 查询文档修改记录
- `document_edit_logs.user_id` - 查询用户修改记录
- `document_edit_logs.created_at` - 按时间查询
- `document_edit_logs.action` - 按操作类型查询

### 日志管理

**保留策略：**
- 查看记录：15天
- 修改记录：15天
- 操作日志：90天（用于安全审计）

**清理机制：**
- 定时任务：每天凌晨3点
- 批量删除：避免性能影响
- 日志记录：便于监控
