# 快速部署指南

## 📦 前置要求

- JDK 17+
- Maven 3.8+
- MySQL 8.0+
- Redis 6.0+
- Node.js 16+
- npm 或 yarn

## 🚀 快速开始

### 1. 数据库初始化

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE wiki_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 使用数据库
USE wiki_db;

# 执行迁移脚本
SOURCE backend/src/main/resources/schema-update.sql;
```

### 2. 配置文件

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/wiki_db?useUnicode=true&characterEncoding=utf8mb4
    username: root
    password: your_password

  redis:
    host: localhost
    port: 6379
    password: # 如果有密码则填写
```

### 3. 启动后端

```bash
cd backend
mvn clean package
java -jar target/wiki-backend.jar
```

或使用开发模式：

```bash
cd backend
mvn spring-boot:run
```

后端将在 `http://localhost:8080` 启动。

### 4. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端将在 `http://localhost:5173` 启动。

## 🔧 配置说明

### 三级权限模型

系统支持三种知识库类型：

1. **COMPANY（公司公开）**
   - 全公司所有员工可见、可编辑
   - 适用于公司级别的共享文档

2. **DEPARTMENT（部门）**
   - 部门内成员可见、可编辑
   - 部门部长拥有管理权限
   - 需要先创建部门并分配用户

3. **PRIVATE（私有）**
   - 仅创建者可见、可编辑
   - 适用于个人笔记和草稿

### 日志管理

系统会自动记录：
- 文档查看记录（保留15天）
- 文档修改记录（保留15天）

定时任务每天凌晨3点自动清理过期日志。

## 👤 初始账号

系统会自动创建管理员账号：

- 用户名: `admin`
- 密码: `admin123`

**⚠️ 首次登录后请立即修改密码！**

## 📝 创建部门

1. 使用管理员账号登录
2. 进入后台管理系统（待开发）
3. 创建部门并指定部门部长
4. 将用户分配到对应部门

## 🔍 验证部署

### 检查后端

```bash
curl http://localhost:8080/actuator/health
```

应返回：
```json
{"status":"UP"}
```

### 检查前端

访问 `http://localhost:5173`，应该看到登录页面。

## 🐛 常见问题

### 1. 数据库连接失败

检查 MySQL 是否启动：
```bash
systemctl status mysql  # Linux
# 或
net start MySQL80       # Windows
```

### 2. Redis 连接失败

检查 Redis 是否启动：
```bash
redis-cli ping
```

应返回 `PONG`。

### 3. 端口被占用

修改配置文件中的端口：
- 后端: `application.yml` 中的 `server.port`
- 前端: `vite.config.js` 中的 `server.port`

### 4. 编译错误

清理并重新编译：
```bash
cd backend
mvn clean install -DskipTests
```
