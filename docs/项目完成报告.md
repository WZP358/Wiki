## ✅ 本次完成的核心功能

### 1. 三级权限模型 🔐

实现了符合企业实际场景的三级权限体系：

| 类型 | 图标 | 可见性 | 编辑权限 | 管理权限 |
|------|------|--------|----------|----------|
| COMPANY | 🏢 | 全公司员工 | 全公司员工 | 创建者 + 系统管理员 |
| DEPARTMENT | 🏛️ | 部门成员 | 部门成员 | 创建者 + 部门部长 + 系统管理员 |
| PRIVATE | 🔒 | 仅创建者 | 仅创建者 | 创建者 + 系统管理员 |

**关键特性：**
- 基于部门的权限隔离
- 部门部长拥有部门知识库管理权
- 系统管理员拥有全局管理权限

### 2. 部门管理体系 🏛️

**数据库设计：**
- 新增 `departments` 表
- 支持部门层级结构（parent_id）
- 支持部门部长设置（manager_id）
- 用户表关联部门（department_id）

**实体实现：**
- [Department.java](backend/src/main/java/com/wiki/app/dept/Department.java)
- [DepartmentRepository.java](backend/src/main/java/com/wiki/app/dept/DepartmentRepository.java)

### 3. 文档操作记录 📝

**查看记录（document_view_logs）：**
- 记录用户ID、用户名、IP地址
- 记录User-Agent（浏览器信息）
- 支持查看时长统计
- **自动保留15天**

**修改记录（document_edit_logs）：**
- 记录操作类型（CREATE/UPDATE/DELETE/PUBLISH）
- 记录修改前后的标题和内容长度
- 记录提交说明（commit_message）
- 记录IP地址
- **自动保留15天**

**实体实现：**
- [DocumentViewLog.java](backend/src/main/java/com/wiki/app/doc/DocumentViewLog.java)
- [DocumentEditLog.java](backend/src/main/java/com/wiki/app/doc/DocumentEditLog.java)
- 对应的Repository类

### 4. 自动清理机制 🧹

**定时任务：**
- [DocumentLogCleanupScheduler.java](backend/src/main/java/com/wiki/app/doc/DocumentLogCleanupScheduler.java)
- 每天凌晨3点自动执行
- 清理超过15天的查看和修改记录
- 使用批量删除，避免性能影响
- 记录清理日志便于监控

**保留策略：**
- 查看记录：15天
- 修改记录：15天
- 操作日志：90天（用于安全审计）

## 🔐 权限检查逻辑

### 知识库权限流程

```
访问请求
  ↓
用户登录？
  ├─ 否 → 拒绝访问
  └─ 是 → 系统管理员？
           ├─ 是 → 允许访问
           └─ 否 → 知识库类型？
                    ├─ COMPANY → 允许访问
                    ├─ DEPARTMENT → 同部门？
                    │                ├─ 是 → 允许访问
                    │                └─ 否 → 拒绝访问
                    └─ PRIVATE → 是创建者？
                                  ├─ 是 → 允许访问
                                  └─ 否 → 拒绝访问
```

### 核心权限方法

- `ensureKbVisible()` - 检查知识库可见性
- `ensureKbEditor()` - 检查知识库编辑权限
- `ensureKbAdmin()` - 检查知识库管理权限
- `canRead()` - 检查文档可读性
- `ensureEditable()` - 检查文档可编辑性

### 1. 数据库迁移


### 2. 启动后端

```bash
cd backend
mvn spring-boot:run
```

### 3. 启动前端

